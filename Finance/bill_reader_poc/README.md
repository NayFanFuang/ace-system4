# ACE Bill Reader — PoC (ระบบอ่านบิล → Payment Voucher)

พิสูจน์แนวคิด: อ่านบิลค่าโทรศัพท์ (AIS / True / DTAC) ที่เป็น **PDF สแกน**
แล้วแปลงเป็น **Payment Voucher (ฟอร์ม PV.03)** อัตโนมัติ ด้วย **AI Vision**

## Pipeline

```
PDF สแกน ──render──► รูปต่อหน้า ──AI Vision──► JSON ต่อเบอร์
        ──compute──► เติม WHT 3% + Net ──fill──► Excel PV.03
```

| ขั้น | ทำอะไร | เครื่องมือ |
|---|---|---|
| 1. render | แตก PDF เป็นรูปรายหน้า | `pymupdf` |
| 2. vision | อ่าน เบอร์ / รอบบิล / ยอดก่อน VAT / VAT 7% | Claude Vision (`anthropic`) |
| 3. compute | คำนวณ WHT 3% และ Net | กฎภาษี |
| 4. fill | กรอกลงเทมเพลต PV.03 | `openpyxl` |

## ค่าที่อ่าน vs ค่าที่คำนวณ

- **อ่านจากบิลจริง:** เบอร์ (Reference No.2), รอบบิล (Bill Cycle), ค่าบริการก่อน VAT, VAT 7%
- **คำนวณเพิ่ม (บิลไม่พิมพ์มา):** `WHT 3% = ยอดก่อน VAT × 3%` · `Net = ยอด + VAT − WHT`
- แปลงปี พ.ศ. → ค.ศ. อัตโนมัติ (2569 → 2026)

## วิธีรัน

```bash
# โหมดจริง (ต้องมี API key)
set ANTHROPIC_API_KEY=sk-...
pip install pymupdf openpyxl anthropic
python bill_reader.py "../ตัวอย่างเอกสาร/7.Bill AIS - TF-04-2026 Item.43-5-9.pdf"

# โหมดเดโม (ไม่มี key → ใช้ผล vision ที่ cache ไว้)
python bill_reader.py "../ตัวอย่างเอกสาร/7.Bill AIS - TF-04-2026 Item.43-5-9.pdf" \
    --cache cache/ais_TF04_item43.json --out output/PV-AIS-TF04-Item43.xlsx
```

> ใน environment นี้ยังไม่มี `ANTHROPIC_API_KEY` จึงรันโหมดเดโมจาก `cache/ais_TF04_item43.json`
> ซึ่งเป็นผลที่อ่านจากบิล AIS จริง (5 หน้า) — pipeline ส่วน render → compute → Excel ทำงานครบ

## หัวกระดาษ — เติมเองอัตโนมัติ (ไม่เหลือ placeholder)

ฟิลด์ที่อ่านจากบิลไม่ได้ ระบบเติมให้เองทั้งหมด (`build_header`):

| ฟิลด์ | ที่มา |
|---|---|
| PV No. / Item | แกะจากชื่อไฟล์ เช่น `TF-04-2026 Item.43` → `TF-04/2026`, `43` |
| Date | วันที่ปัจจุบัน (`DD-Mon-YYYY`) |
| Project / ผู้ขอเบิก / ผู้จัดทำ | ค่าตั้งต้นใน `DEFAULTS` ที่หัวสคริปต์ (แก้ได้) |

## ข้อจำกัด / สิ่งที่ต้องต่อยอด

- `DEFAULTS` (Project/ผู้เบิก) เป็นค่ามาตรฐานที่เดาให้ — ควรยืนยัน/ผูกกับระบบจริงภายหลัง
- รองรับ ≤ 12 รายการต่อ 1 PV (ตามแถวของเทมเพลต) — เกินต้องแตกหลายใบ
- ทดสอบกับ AIS แล้ว; True/DTAC/NT ใช้ prompt เดียวกันได้แต่ควร verify ฟิลด์เบอร์/รอบบิล
- ยังไม่ได้ตรวจสอบยอดข้ามกับ Outstanding/ยอดยกมา — ตอนนี้คิดเฉพาะรอบปัจจุบัน
